# 微信支付V3 开发文档

## 一、项目概述

基于当前官网项目（Vue3 + Naive-UI + SpringBoot3 + MyBatis-Plus），集成微信支付V3版本，实现套餐订阅功能。

### 1.1 功能模块

| 模块 | 功能描述 |
|------|----------|
| 套餐管理 | 创建/展示套餐（Free/Pro/Pro+/Ultra） |
| 订单管理 | 创建订单、查询订单、订单列表 |
| 支付模块 | PC扫码支付（Native）、支付回调、状态轮询 |
| 个人中心 | 我的订单列表 |

### 1.2 技术栈

- **前端**：Vue3 + Naive-UI + Vite + Axios
- **后端**：SpringBoot 3 + MyBatis-Plus + MySQL
- **支付SDK**：wechatpay-java（微信支付V3官方SDK）

## 三、数据库设计

### 3.1 套餐表（plan）

```sql
CREATE TABLE `plan` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `name` VARCHAR(50) NOT NULL COMMENT '套餐名称',
    `code` VARCHAR(20) NOT NULL UNIQUE COMMENT '套餐编码(FREE/PRO/PRO_PLUS/ULTRA)',
    `price` DECIMAL(10,2) NOT NULL COMMENT '价格（元）',
    `original_price` DECIMAL(10,2) COMMENT '原价',
    `credits` INT NOT NULL COMMENT 'Credits数量',
    `duration` INT NOT NULL DEFAULT 30 COMMENT '有效期（天）',
    `features` TEXT COMMENT '功能特性（JSON）',
    `sort` INT DEFAULT 0 COMMENT '排序',
    `status` TINYINT DEFAULT 1 COMMENT '状态：0-下架 1-上架',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='套餐表';
```

### 3.2 订单表（order）

```sql
CREATE TABLE `order` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `order_no` VARCHAR(32) NOT NULL UNIQUE COMMENT '订单号',
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `plan_id` BIGINT NOT NULL COMMENT '套餐ID',
    `plan_name` VARCHAR(50) NOT NULL COMMENT '套餐名称',
    `amount` DECIMAL(10,2) NOT NULL COMMENT '支付金额（元）',
    `status` TINYINT DEFAULT 0 COMMENT '状态：0-待支付 1-已支付 2-已取消 3-已退款',
    `pay_type` VARCHAR(20) COMMENT '支付方式：WECHAT_NATIVE',
    `transaction_id` VARCHAR(64) COMMENT '微信支付订单号',
    `pay_time` DATETIME COMMENT '支付时间',
    `expire_time` DATETIME COMMENT '订单过期时间',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX `idx_user_id` (`user_id`),
    INDEX `idx_order_no` (`order_no`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';
```

### 3.3 用户套餐表（user_plan）

```sql
CREATE TABLE `user_plan` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `plan_id` BIGINT NOT NULL COMMENT '套餐ID',
    `plan_name` VARCHAR(50) NOT NULL COMMENT '套餐名称',
    `credits` INT NOT NULL COMMENT '总Credits',
    `used_credits` INT DEFAULT 0 COMMENT '已使用Credits',
    `start_time` DATETIME NOT NULL COMMENT '生效时间',
    `end_time` DATETIME NOT NULL COMMENT '到期时间',
    `order_no` VARCHAR(32) NOT NULL COMMENT '关联订单号',
    `status` TINYINT DEFAULT 1 COMMENT '状态：0-已过期 1-生效中',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户套餐表';
```

## 四、后端接口设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/plan/list` | GET | 获取套餐列表 |

| 接口 | 方法 | 说明 |
|------|------|------|

| `/api/pay/query/{orderNo}` | GET | 主动查询支付结果 |

## 五、接口详细设计

### 5.1 创建订单

**请求**
```
POST /api/order/create
Content-Type: application/json
Authorization: Bearer {token}

{
    "planId": 2
}
```

**响应**
```json
{
    "code": 200,
    "data": {
        "orderNo": "202603291234567890",
        "planName": "Pro",
        "amount": 10.00,
        "expireTime": "2026-03-29 12:30:00"
    }
}
```

### 5.2 发起Native支付

**请求**
```
POST /api/pay/native
Content-Type: application/json
Authorization: Bearer {token}

{
    "orderNo": "202603291234567890"
}
```

**响应**
```json
{
    "code": 200,
    "data": {
        "codeUrl": "weixin://wxpay/bizpayurl?pr=xxxxx",
        "orderNo": "202603291234567890"
    }
}
```

> 前端使用 `qrcode` 库将 `codeUrl` 生成二维码展示

### 5.3 轮询订单状态

**请求**
```
GET /api/order/status/{orderNo}
Authorization: Bearer {token}
```

**响应**
```json
{
    "code": 200,
    "data": {
        "orderNo": "202603291234567890",
        "status": 1,
        "statusText": "已支付"
    }
}
```

**状态码说明**
| status | 说明 |
|--------|------|
| 0 | 待支付 |
| 1 | 已支付 |
| 2 | 已取消 |
| 3 | 已退款 |

### 5.4 我的订单列表

**请求**
```
GET /api/order/list?page=1&pageSize=10
Authorization: Bearer {token}
```

**响应**
```json
{
    "code": 200,
    "data": {
        "total": 5,
        "list": [
            {
                "orderNo": "202603291234567890",
                "planName": "Pro",
                "amount": 10.00,
                "status": 1,
                "statusText": "已支付",
                "payTime": "2026-03-29 12:05:30",
                "createTime": "2026-03-29 12:00:00"
            }
        ]
    }
}
```

### 5.5 支付回调通知

**微信服务器请求**
```
POST /api/pay/notify
Content-Type: application/json
Wechatpay-Signature: ...
Wechatpay-Timestamp: ...
Wechatpay-Nonce: ...
Wechatpay-Serial: ...

{密文数据}
```

**处理逻辑**
1. 验证签名
2. 解密通知数据
3. 更新订单状态为已支付
4. 创建用户套餐记录
5. 返回成功响应

**响应**
```json
{
    "code": "SUCCESS",
    "message": "成功"
}
```

### 6.3 轮询逻辑

```javascript
// 支付页面轮询逻辑（伪代码）
const pollInterval = 3000  // 3秒
const maxPollTime = 300000 // 最长5分钟

let timer = setInterval(async () => {
    const res = await checkOrderStatus(orderNo)
    if (res.status === 1) {
        // 支付成功，跳转结果页
        clearInterval(timer)
        router.push('/pay-result?orderNo=' + orderNo)
    }
}, pollInterval)

// 超时处理
setTimeout(() => {
    clearInterval(timer)
    // 提示用户订单超时
}, maxPollTime)
```

## 八、注意事项

### 8.1 安全要求

- 回调接口必须验证微信签名
- 私钥文件不能提交到代码仓库
- 回调地址必须使用HTTPS

### 8.2 幂等性处理

- 回调可能多次，需根据订单号做幂等判断
- 已处理的订单直接返回成功

### 8.3 订单超时

- 订单创建后设置15-30分钟有效期
- 定时任务关闭超时未支付订单

### 8.4 内网穿透（开发环境）

开发环境可使用 natapp/ngrok 等工具进行内网穿透：

```bash
# natapp示例
./natapp -authtoken=your_token
# 获得公网地址如: http://xxx.natappfree.cc
```
